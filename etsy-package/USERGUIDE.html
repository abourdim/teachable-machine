The two kinds of files

KindWho sees itWhere it goes
Buyer-facingThe customer who pays on EtsyBundled into the ZIP, or used as listing images
Seller-onlyYou (and anyone running this shop)Stays on disk / in this repo — never uploaded
Never paste content from seller-only/ into the Etsy listing itself. It's strategy, legal, or video-production material — not buyer copy. The one exception is the title, description, and tags from ETSY_LISTING.md, which are explicitly designed to be pasted.

File-by-file reference

🛠 Tooling

build-package.js tool

What
Node script that renders the 6 printable HTML templates to PNG via Playwright, renders the 7 Etsy listing mockups, and zips everything into TeachableMicrobit-v<version>.zip.
When
Every release. Run from the repo root: 
node etsy-package/build-package.js
# or
npm run build:etsy
Outputs
etsy-package/output/*.png, etsy-package/TeachableMicrobit-v<version>/, etsy-package/TeachableMicrobit-v<version>.zip.
Versioning
Edit the VERSION constant at the top of the file when you bump.

📜 Buyer license

LICENSE.txt ships in ZIP

What
The end-user license shipped inside the ZIP. Single-user / single-classroom terms. Covers the app + printables. Eleven numbered clauses including Updates, Refunds, Privacy, and Proof of License.
When
Automatically copied into the ZIP by build-package.js.
Edit carefully
Changing this changes what buyers agree to. Keep clause 4 (Updates) in lockstep with the Etsy listing's "Will it get updates?" FAQ — divergence has been flagged as a review risk.

🖨 Printables — shipped in the ZIP under printables/

All six are standalone HTML files designed to be rendered as PNGs by build-package.js. The HTML sources ship too, so teachers can tweak them.

quickstart-card.html A4 portrait

What: 5-step setup card (flash firmware → open index.html → Chrome/Edge → connect BLE → play).

When: The first thing buyers see on unzip. Also the cover image of the "Printables" bundle advertised on the listing.

shortcuts-cheatsheet.html A4 landscape

What: Single-page flow reference (Train → Load URL → Start Webcam → Connect → Start Sending), BLE state cheat sheet, TEST button tips.

When: Teachers print and pin next to the classroom PC.

classroom-poster.html A3 portrait

What: "We Control Robots with Code!" poster in 5 big kid-safe steps.

When: Teachers print on A3 for classroom walls. Doubles as a strong visual for the listing cover.

lesson-plan-template.html A4 portrait

What: Editable 45-minute lesson-plan template + one ready-to-teach sample ("Hot or Not? The Sensor Detective") with a rubric.

When: Sell the "teacher-friendly" angle. Print, fill in by hand or in Canva, hand out.

sticker-sheet.html A4 portrait

What: 30 circular badges — "I Trained an AI!", "I Connected BLE!", "My Micro:bit Listens!" and other achievement stickers.

When: Teachers print on sticker paper (Avery 22807 or similar) for student rewards. Great "bonus" line in the Etsy description.

README-quickstart.html A4 portrait

What: Polished buyer welcome page ("Thank you for buying…" + first 3 things to do).

When: Ships alongside README.md in the ZIP so buyers see a printable welcome, not just a plain text file.

🖼 Listing mockups

etsy-listing-mockups.html source

What: Single HTML file containing 7 <div class="mockup"> elements, each 2000×1500 px. Rendered by build-package.js into output/etsy-mockup-1.png … etsy-mockup-7.png.

The seven mockups (in upload order):

  1. Hero — laptop + webcam + BBC micro:bit V2 in frame, "Teach • Trigger • Control" overlay
  2. Teachable Machine model-training screen — three classes with confidence bars
  3. Gesture → LED reaction GIF — hand pose on the left, micro:bit icon on the right
  4. Classroom scene — kids teaching the AI at a shared laptop
  5. TEST button close-up — the one-click BLE sanity check
  6. Mobile view — Chrome sized down to show the single-page flow
  7. Bundle / bonus card — "What's in the ZIP" manifest + lifetime-updates badge

When: Upload as the 7 listing images on Etsy in this order. Etsy shows the first image as the search thumbnail.

Crop caveat: 2000×1500 is 4:3. Etsy's first-photo square crop will chop 500 px off the sides — verify the product name and the compat badge survive the center-square crop before publishing.

🔒 seller-only/ — strategy, legal, video

All files under seller-only/ are excluded from the buyer ZIP by build-package.js. Don't manually zip them in.

ETSY_LISTING.md / .html seller-only

What: The full business playbook. Title, description, pricing (single-user $14.99, classroom $34.99, school site $249, district $599+), all 13 tags, paste-ready social copy, Pinterest plan, video scripts, 10 tier-2 product ideas. The .html twin has a copy-to-clipboard button on every paste-ready block.

When: Your source of truth when you're typing into the Etsy "Create listing" form.

ETSY_PUBLISH_GUIDE.html seller-only

What: 16-step launch checklist with progress tracking (localStorage) and status pills. Covers Etsy shop setup → first Pinterest pin.

When: Open this first on launch day. Tick off each step. Progress persists across reloads.

etsy-playbook.html seller-only

What: 60-second video script + shot list for the Etsy listing video. English-only — this product is monolingual.

When: Day you film. Keep open on a second screen while recording.

ETSY-1MIN-PLAYBOOK.md seller-only

What: Markdown twin of etsy-playbook.html. Easier to diff in Git when you iterate on the script.

When: Interchangeable with the HTML — use whichever is handier.

LICENSE-SITE seller-only

What: Legal text for the School Site License tier ($249, up to 30 teachers at one school, unlimited students).

When: When you create the higher-tier listing on Etsy. Rename to LICENSE inside that tier's ZIP before shipping.

SITE_LICENSE_CERTIFICATE.html seller-only

What: Per-sale certificate template. Fill in school name, order date, number of seats → print to PDF → email to the buyer.

When: After each School Site License order on Etsy.

TODO.md seller-only

What: Pre-launch + post-launch manual checklist — things the build pipeline can't automate (real micro:bit photo, square-crop sanity check, compat infographic, listing video, post-launch conversion monitoring).

When: Review before hitting Publish on Etsy, and weekly for the first month after.

The full Etsy workflow, in order

1. Prep (once per release)

  1. Bump VERSION in build-package.js and package.json.
  2. Run npm run build:etsy — produces the ZIP and the 7 mockup PNGs.
  3. Open seller-only/ETSY_LISTING.md, update anything that changed (features list, pricing, version number in the description).
  4. Preview etsy-listing-mockups.html in Chrome; verify all 7 mockups render with current branding.

2. Record the video (once)

  1. Open seller-only/etsy-playbook.html on a second screen. Record in OBS following the shot list.
  2. Export a 60-second MP4, ≤100 MB, 1080p or higher (Etsy's limit).

3. Create the listing (launch day)

  1. Open seller-only/ETSY_PUBLISH_GUIDE.html in Chrome. Tick each step as you go.
  2. Create the Etsy listing. Paste title, description, tags from seller-only/ETSY_LISTING.md.
  3. Upload the 7 mockup PNGs from output/etsy-mockup-1.png … etsy-mockup-7.png in numeric order (Etsy uses the first one as the thumbnail).
  4. Upload the video from step 6.
  5. Attach the digital file: TeachableMicrobit-v<version>.zip.
  6. Set pricing from the ETSY_LISTING.md pricing table.
  7. Publish.

4. After each sale

  1. Single-user / classroom: nothing to do — Etsy delivers the ZIP automatically. The buyer sees README-quickstart.html + quickstart-card.pdf first.
  2. School Site License: fill in seller-only/SITE_LICENSE_CERTIFICATE.html, print to PDF, email to the buyer. Log the order in your own tracker.

5. Ongoing maintenance

  1. When the app ships a new feature, edit ETSY_LISTING.md, bump the version, re-run npm run build:etsy, and update the Etsy listing's description + attached ZIP.
  2. Keep old ZIPs tracked in Git (the .gitignore exemption keeps etsy-package/TeachableMicrobit-v*.zip in history) so you can always recover the exact artifact any past buyer received.

Rules of thumb

🚫 Don'ts

  • Never paste pricing strategy from ETSY_LISTING.md into the public listing description — that's your playbook, not the pitch.
  • Never zip seller-only/ manually into the buyer ZIP. build-package.js intentionally excludes it.
  • Never let LICENSE.txt clause 4 drift from the Etsy listing's "Will it get updates?" FAQ — buyers will call it out in reviews.
Always rebuild the ZIP after editing anything in docs/, assets/, js/, or the root app files. The ZIP is tracked in Git, so the buyer-facing artifact is always reproducible.
Rule of thumb: if a file has LISTING, PUBLISH, PLAYBOOK, SITE, or CERTIFICATE in its name — it's seller-only.
